# Создание визуальных блоков и работа с плагинами в Spraute Engine

Spraute Engine поддерживает полностью кастомизируемую среду визуального программирования. Вы можете не только использовать встроенные блоки, но и создавать свои собственные, объединять их в плагины и делиться ими с другими пользователями. 

Система визуальных блоков работает поверх языка `.spr` и использует специальные комментарии `#\`, чтобы превратить ваш код в красивые визуальные элементы (блоки).

---

## 1. Правила построения визуального блока

Основой для каждого пользовательского блока служит обычный текстовый файл `.spr`. Вы можете размещать такие файлы в своих плагинах. Чтобы движок распознал блок, используйте синтаксис `#\`.

### Структура определения блока

Каждый блок состоит из трех секций:
1. **Заголовок и Метаданные:** Название, цвет, форма, категория.
2. **[UI] и [DYNAMIC_UI] (Внешний вид):** Как блок выглядит и как меняется (мутирует) при выборе параметров.
3. **[CODE_GEN] (Генерация кода):** Какую строчку кода на языке `.spr` сгенерирует этот блок.

### Пример сложного блока: «Взгляд НИПа»

```spraute
#\ block: npc_look
#\ category: NPC
#\ color: #0ea5e9
#\ shape: statement
#\
#\ [UI]
#\ // Заголовок (Субъект): Всегда начинается с того, КТО делает действие
#\ row: "НИП" [npc: dropdown_npc] "смотреть" [mode: dropdown(один раз: once, вечно: loop, перестать: stop)]
#\
#\ [DYNAMIC_UI]
#\ // Логика цели (динамический слот): меняется в зависимости от выбора цели
#\ if mode != "stop":
#\   row: "на" [target_type: dropdown(позиция: pos, НИП: npc, игрок: player, кастомное: custom)]
#\   
#\   if target_type == "pos":
#\     row: "x" (x: number: 0) "y" (y: number: 64) "z" (z: number: 0)
#\   
#\   if target_type == "npc":
#\     row: "цель НИП:" [target_npc: dropdown_npc]
#\   
#\   if target_type == "player":
#\     row: "игрок" [player_type: dropdown(по имени: name, первый/ближайший: nearest, системная переменная: var)] (player_val: text: "player")
#\   
#\   if target_type == "custom":
#\     // Слот для вставки другого блока-значения (например, переменной)
#\     input: custom_target (type: value) "выражение"
#\
#\ [CODE_GEN]
#\ // Генерация итогового кода
#\ if mode == "stop": 
#\   return `${npc}.stopLook();\n`
#\ 
#\ let targetStr = "null";
#\ if target_type == "pos": targetStr = `${x}, ${y}, ${z}`;
#\ if target_type == "npc": targetStr = target_npc;
#\ if target_type == "player":
#\   if player_type == "name": targetStr = `getPlayer("${player_val}")`;
#\   if player_type == "nearest": targetStr = `getNearestPlayer(${npc})`;
#\   if player_type == "var": targetStr = player_val;
#\ if target_type == "custom": targetStr = custom_target;
#\
#\ if mode == "once": return `${npc}.lookAt(${targetStr});\n`
#\ if mode == "loop": return `${npc}.alwaysLookAt(${targetStr});\n`
```

### Типы полей и синтаксис
- **[ текст: тип ]** - выпадающие списки (`dropdown`).
- **( текст: тип: значение )** - текстовые или числовые поля ввода.
- **`input: name (type: wrapper)`** - создание C-образного блока, внутрь которого можно поместить другие блоки (например, для создания своих конструкций `if / else` или циклов).

## 2. Плагины и Менеджер Плагинов

Вы можете создавать плагины для движка. Плагины управляются во вкладке **Управление -> Плагины**.

### Что может содержать плагин?
Плагин может состоять из:
1. **Только `.spr` библиотек:** Набор пользовательских визуальных блоков (с синтаксисом `#\`) или скриптов.
2. **Только JS плагина:** Расширение функционала редактора или движка (внутренние изменения).
3. **Комбинированного плагина:** И `.spr` блоки, и JS функционал.

### Как создать плагин
1. Откройте **Управление -> Плагины**.
2. Нажмите **«Создать плагин»**. Задайте ему имя (например, `CombatSystem`).
3. В папке проекта появится структура плагина `plugins/CombatSystem/`.
4. Внутрь папки `blocks/` вы можете поместить свои `.spr` файлы с описанием новых визуальных блоков.
5. При загрузке проекта движок автоматически просканирует эти файлы и добавит ваши новые блоки в палитру.

## 3. Импорт скриптов и Визуальный Редактор

Чтобы НИПы, переменные и функции подгружались в выпадающие списки (например, `dropdown_npc`), движок сканирует не только текущий открытый скрипт, но и **все импортированные скрипты**.

### Кнопка «Импорт» в Визуальном Редакторе
В визуальном редакторе добавлена функция импорта других файлов `.spr`. 
1. В верхней панели редактора нажмите кнопку **«Импортировать скрипт»**.
2. Выберите нужный файл из вашего проекта или из плагинов.
3. Теперь все НИПы, функции и переменные, определенные в импортируемом файле, станут доступны в визуальных блоках (например, в списках `цель НИП`).

## 4. Конструкции if / else (Вложения)

Чтобы создать блок с вложенностью (wrapper), используйте форму `shape: wrapper`.

```spraute
#\ block: my_if
#\ category: Логика
#\ color: #f59e0b
#\ shape: wrapper
#\
#\ [UI]
#\ row: "Если"
#\ input: condition (type: value) // Сюда вставляется условие (например, 1 < 2)
#\ row: "То выполнить:"
#\ input: DO (type: statement) // Сюда вставляются блоки действий
#\
#\ [CODE_GEN]
#\ let condCode = condition || "false";
#\ let doCode = DO || "";
#\ return `if (${condCode}) {\n${doCode}}\n`;
```
Таким образом, пользователи могут неограниченно вкладывать блоки друг в друга и конструировать сложнейшую логику исключительно визуально.
